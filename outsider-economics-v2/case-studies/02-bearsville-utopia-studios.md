# Bearsville / Utopia Studios — The room where the record happened is now the node.

Woodstock, New York isn't a theme park. It's a town full of people who've seen hype die in a ditch. Utopia sits in that reality — a recording campus with history — and the job was never "make it pretty." The job was **make it run like a regional media company**.

## The town / the place

Bearsville / Woodstock belt. Tourists pass through. Artists stay. The studio is a physical asset most towns don't have — **live rooms, isolation, credibility**.

## The problem

Studios make *files*. Cities make *pipelines*. If your files don't have a path to audience, revenue, and archive, you're renting time instead of building equity.

The leak: **beautiful audio trapped in hard drives** while distribution still defaults to platforms that tax attention and pay in exposure.

## What we built

**Segmented production network** — VLANs so public WiFi, client access, and the recording rig aren't the same blast radius.

**Ubiquiti WiFi** — outdoor coverage, mesh extensions, a path to **captive portal** marketing that feeds the directory instead of a third-party ad stack.

**NAS archive** — mirror from the Mac Studio, retention rules that don't punish you for deleting a bad take locally while keeping the masters cold.

**Multi-camera live path** — wireless video into switch / record, then out to Restream-class publishing — the live stack ties to Studio C ops (see repo `/admin/studio-c` and production jobs).

Source material: Utopia Video routing, BMG network architecture, Studio C tech sheets, BMG + Woodstock ecosystem maps.

## The math

Quotes in the packet land in four bands — **WiFi hardware in the hundreds**, **NAS builds from ~$1.4k–$2k+** depending on SSD tier, **day rates** for crew and gear that have to compete with "just shoot it on a phone." The winning comparison isn't phone vs. cinema — it's **margin kept per job** when you're not renting every middle layer.

## What it means

**A recording studio can be a regional media company** if you treat the building like a **node** — network, storage, programming, and client-facing rails — not like a rental calendar with a logo.
