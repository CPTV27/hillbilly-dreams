# Chapter 4: The Task Board Economy

## Overview

The task board is the core infrastructure of coordination economics. It consists of three columns — TASKS AVAILABLE, IN PROGRESS, COMPLETED — and functions as the central nervous system of a community economy. This chapter covers how task boards create coordination, how they function as a monetary system, and how governance emerges through visibility rather than hierarchy.

The task board replaces both the market (for resource allocation) and the bank (for currency issuance). Tasks posted are currency issued. Tasks completed are value created. The board itself is the ledger, the marketplace, and the governance mechanism in a single artifact.

Volume 2, Chapter 4, provides industry-specific task board templates for hospitality, music, and agriculture communities. Volume 3 describes federated task board systems operating across a multi-county corridor.

---

## Tasks as Currency Issuance

Traditional monetary systems issue currency through debt creation. Banks loan money into existence, requiring interest payments that extract value from communities. Task board systems issue currency through labor commitment. Communities create purchasing power by pledging productive time.

| System | Currency Creation | Outcome |
|---|---|---|
| Debt-based | Bank → Loan → Money Supply → Interest Obligation | Value extraction (must repay more than created) |
| Labor-based | Community → Task Commitment → Hour Credits → Value Creation | Wealth retention (creates exactly the value it represents) |

Debt-based money requires paying back more than was created, guaranteeing scarcity. Labor-based currency creates exactly the value it represents, enabling abundance.

## Dynamic Hour Valuation

Task board systems transcend traditional time banks through dynamic valuation. Hour values fluctuate based on community needs rather than external market rates. This creates responsive monetary policy at the community scale.

### Static Time Banking (Traditional)
1 hour = 1 hour, regardless of task, skill, or need.

### Dynamic Task Board Economy
1 hour = variable value based on:
- Community priority multipliers
- Skill rarity within the network
- Task completion urgency
- Member capacity and availability

### The Task Board Monetary System

| Monetary Function | Task Board Equivalent |
|---|---|
| Currency | Hours committed to task completion |
| Reserve | Member skills and available time |
| Monetary policy | Task categorization and priority multipliers |
| Central bank | Collective governance through task board meetings |
| Interest rate | Community priority adjustments to hour values |

## Labor as Monetary Policy

In traditional economies, central banks adjust interest rates to influence economic activity. In task board economies, communities adjust task priorities and hour multipliers to direct labor toward community needs.

| Traditional Tools | Task Board Tools |
|---|---|
| Interest rate adjustments | Priority multiplier adjustments |
| Money supply controls | Skill requirement specifications |
| Reserve requirements | Hour contribution expectations |
| Open market operations | Task categorization changes |

When a community needs more childcare coordination, it increases childcare task values from 1.0x to 1.4x base rate. This naturally incentivizes members to offer more childcare hours. When maintenance capacity is sufficient, those multipliers decrease, directing labor toward higher-need areas.

This creates responsive resource allocation without central planning. The task board automatically directs labor toward community priorities through decentralized price signals.

## The Five Essential Task Categories

Successful task board communities organize around five core categories. The distribution balances individual needs with collective infrastructure:

### The 40-25-20-10-5 Rule

| Category | Allocation | Examples |
|---|---|---|
| Direct Service Exchange | 40% | Cleaning, cooking, childcare, pet care, tutoring, music lessons, bookkeeping, tax prep |
| Community Infrastructure | 25% | Tool library maintenance, community garden, event coordination, task board administration |
| External Income Generation | 20% | Collective freelance projects, community business development, grant writing |
| Learning and Development | 10% | Skill sharing workshops, training, research, documentation |
| Governance and Coordination | 5% | Conflict resolution, strategic planning, new member onboarding |

Communities that over-emphasize direct service exchange stagnate. Communities that over-emphasize infrastructure become bureaucratic. The 40-25-20-10-5 distribution has proven optimal across diverse communities studied to date.

## Hour Tracking and Verification

Task board systems require lightweight accountability without bureaucratic overhead. Three verification levels handle the full range of task complexity:

### Verification Tiers

| Tier | Coverage | Method | Use Case |
|---|---|---|---|
| Self-reporting | 80% of tasks | Members log own hours; monthly reconciliation catches discrepancies | Routine tasks, ongoing contributions |
| Peer verification | 15% of tasks | Task recipient confirms completion and quality | Direct service exchanges, external projects |
| Community verification | 5% of tasks | Multi-member review for complex or high-impact work | Infrastructure projects, governance tasks, resource allocation |

Self-reporting works for the majority of tasks because community trust and reputation provide natural accountability. Peer verification catches honest estimation errors. Community verification provides oversight for tasks affecting the whole group.

## Conflict Resolution Protocols

Coordinated communities generate conflicts. The following four-tier escalation system maintains relationships while ensuring resolution:

### Tier 1: Direct Conversation (80% of conflicts)
- Affected parties communicate directly
- 24-48 hour timeline for initial conversation
- Focus on needs and solutions, not blame

### Tier 2: Peer Mediation (15% of conflicts)
- Trusted community member facilitates
- Structured process with agreed ground rules
- Outcome documented for community learning

### Tier 3: Community Circle (4% of conflicts)
- Full community discussion using restorative justice principles
- All voices heard through structured process
- May result in policy changes or process improvements

### Tier 4: Federation Arbitration (1% of conflicts)
- External cooperative mediator for unresolvable disputes
- Available only for federated communities (see Chapter 6)
- Binding arbitration focused on relationship preservation

### Conflict Prevention Systems
- Regular check-ins on member satisfaction and capacity
- Clear task descriptions and expectations before work begins
- Training in nonviolent communication techniques
- Written community agreements about respectful engagement

## Governance Through Task Allocation

Task board communities govern themselves through resource allocation rather than meeting-based democracy. Members vote with their time commitments, creating organic consensus.

### Traditional Cooperative Governance
1. Members debate priorities in meetings
2. Vote on resource allocation proposals
3. Designated leaders implement decisions
4. Results evaluated at next meeting cycle

### Task Board Governance
1. Community needs emerge through member requests
2. Tasks created with appropriate hour values and multipliers
3. Members self-select into tasks based on capacity and interest
4. Resource allocation happens through distributed individual choice

This eliminates the meeting fatigue that destroys many cooperatives while ensuring democratic resource allocation through individual agency rather than collective coercion.

## Multiplier Adjustment Process

Dynamic hour valuation requires clear governance around how priority multipliers are set and adjusted:

### Automatic Monthly Adjustment
- Tasks with less than 50% completion rate receive +0.2x multiplier
- Tasks with greater than 90% completion rate decrease by -0.1x multiplier
- System self-corrects based on actual community response

### Quarterly Strategic Adjustment
- Community review of priority alignment with values and goals
- Democratic vote on major multiplier changes (greater than 0.5x shifts)
- Seasonal adjustments for changing community needs

### Emergency Adjustment Authority
- Designated coordinator can temporarily adjust multipliers for crisis response
- Changes automatically expire after 30 days unless ratified by community
- Used for sudden needs: illness, housing crisis, external deadlines

### Annual Comprehensive Review
- Complete evaluation of multiplier effectiveness
- Comparison with external market rates for calibration
- Strategic planning for next year's priority areas

## The Emergence Effect

Task board governance leverages emergence — complex systems arising from simple rules.

**Simple Rules:**
- Create tasks when needs arise
- Contribute hours based on capacity and interest
- Adjust hour values based on community response

**Emerging Complexity:**
- Organic specialization as members find preferred tasks
- Natural rotation as people develop new skills
- Automatic resource allocation toward highest-value activities

No central planning required. The system self-organizes.

## Infrastructure Blueprint

### Phase 1: Whiteboard and Notebooks (Weeks 1-4)

Start with physical tools. This forces focus on coordination principles before technology investment.

**Essential infrastructure:**
- Physical task board with movable cards (whiteboard, corkboard, or refrigerator)
- Shared notebook for hour tracking (one page per member)
- Simple meeting agenda template
- 3-5 basic task categories

**Week 1 setup:**
1. Create three columns: AVAILABLE / IN PROGRESS / COMPLETED
2. Each member contributes 2-3 initial tasks they need done
3. Each member commits to 5-10 hours for the first month
4. Schedule weekly check-ins

**Success metrics for first month:**
- 80% of initial tasks completed within 30 days
- Every member both offers and receives task value
- Zero major conflicts requiring external mediation

### Phase 2: Spreadsheet Coordination (Months 2-6)

As communities grow beyond 8-10 active members, shared spreadsheets enable better tracking.

**Google Sheets template (three tabs):**
- **Tasks**: Description, category, estimated hours, assigned member, completion date
- **Hours**: Member name, monthly hours committed, hours completed, running balance
- **Members**: Contact info, skills offered, availability preferences, hour rates

### The Monthly Reset

Every month, reset hour balances to zero and redistribute any surplus. This prevents hour accumulation from creating inequality while maintaining accountability for current commitments.

## Implementation

### Step 1: Gather Founding Circle
Identify 5-8 people interested in coordination. Collect 3-5 task needs from each person.

### Step 2: Design Initial Board
Create three columns. Write each collected task on a separate card. Organize into 3-4 basic categories. Prepare hour tracking method.

### Step 3: Host Launch Meeting (90 minutes)
- Review task board concept and community goals (20 min)
- Each member claims 2-3 tasks to complete (30 min)
- Each member posts 1-2 tasks they need done (20 min)
- Set hour commitments and next meeting date (20 min)

### Step 4: Complete First Task Exchanges
Run the system for two weeks before the first monthly review. Track every hour. Note what works and what does not.

### Step 5: Conduct Monthly Review
Evaluate completion rates. Adjust multipliers if needed. Add new task categories. Onboard new members.
