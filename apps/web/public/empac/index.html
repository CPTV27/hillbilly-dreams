Section 3

The capture process — how a room becomes data

Field capture (today's actions, ~10 min/room)

  1. iPhone Pro + Polycam app. Walk the room, sweep ceiling-floor-walls. Output: .obj mesh + .mtl + textures/
  2. Phone-camera capture video (the .MOV files) — recording the human-eye reference for the mesh
  3. Ambisonic audio capture — sweeps + impulse responses + ambient room recording. (Or, with iPhone 14 Pro+, the 4-channel spatial audio is already baked into the .MOV — no separate rig needed.)
  4. Metadata: room name, date, capture conditions (HVAC on/off, audience present/absent)

Cloud ingest (what S2PX does next)

  1. Hot-folder drop or direct upload to S2PX bucket
  2. S2PX registers a Scan record (project, room, capture date, GPS, paths)
  3. Mesh post-processing (decimation if >50MB, normal recompute, hole-fill)
  4. Audio post-processing (peak normalize, noise-floor characterize, IR convolution)
  5. Lyra layer: LMIR fingerprint computed, RT60 + frequency response logged, spatial-audio export rendered
  6. Customer portal updates: scan available for review, acoustic report generated, BIM annotations queued
