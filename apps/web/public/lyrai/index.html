01What Lyrai is

Lyrai is an acoustic-twin platform: customers capture a real-world space — a concert hall, a recording studio, a heritage site, an architectural interior — and Lyrai returns both a high-fidelity 3D model and a measurable acoustic signature of that space. The output is usable in Unity, Unreal, Wwise, FMOD, BIM tools, and standalone web viewers.

Beneath the acoustic IP — the LiMIR database (LiDAR · Material · Impulse Response), the Smart Acoustic Capture Bayesian engine, the Stony Brook research lineage — Lyrai runs on a multi-tenant platform layer that has been in production for over a year, with hundreds of projects through the catalog and a working customer-facing app. The platform layer is the unfair advantage: most acoustic-twin proposals on the market today are 18 months of operational engineering away from being able to ship anything to a customer. Lyrai is not.

Platform stack

Frontend — React 19 + Vite 6, Tailwind CSS 4, in-browser 3D viewer (Three.js), runs on phone and desktop with no plugin

03Spatial data — already shipping

Once a scan lands, the customer portal provides the full set of post-capture capabilities customers expect from a modern acoustic-twin service:

  • 3D viewer in-browser — Three.js based, runs on phone or desktop, no plugin, full pan / orbit / zoom
  • Annotation overlay — pin notes to mesh coordinates, share with collaborators, tracked per project
  • Asset registry — every scan tied to a project, every project to a customer, every customer to a tenant; full audit trail
  • BIM annotation export — for AEC customers, scan deliverables route into Revit / Rhino workflows
  • Public REST API — endpoints at /api/scans, /api/scans/:id/mesh, /api/scans/:id/textures, /api/projects/:id/scans. Authenticated via tenant API keys. Lyrai-acoustic services consume the same API as third-party customer integrations.

04The audio layer — Lyrai's defining build

Where Lyrai differentiates is the audio layer. The platform handles geometry, texture, viewer, API, billing, and tenancy. The acoustic-intelligence work — what Lyrai actually does — sits on top.

The build is well-scoped because every component has a clear interface to the platform underneath:

  • Build Spatial-audio extraction service. Modern iPhone Pro video files contain 4-channel APAC (Apple Positional Audio Codec) tracks embedded with the visual frames — full ambisonic-style room capture with no separate microphone rig. A small extraction service demuxes the APAC track, normalizes it, and routes it into the LiMIR pipeline.
  • Connect LiMIR fingerprint integration. The platform job queue calls the LiMIR service on every new scan that has audio attached. Result is stored on the Scan record as acoustic_profile, available via the public API.
  • Connect RT60 + frequency-response analysis. Smart Acoustic Capture's Bayesian engine consumes the room geometry and audio, returns a structured analysis. Result lives on the Scan record.
  • Build Customer portal: acoustic overlay. The 3D viewer extends to render the acoustic visualization layer — heat maps, propagation cones, hot spots — on top of the mesh.
  • Build Spatial-audio export. Customers can download the room's acoustic signature as an ambisonic file Unity, Wwise, FMOD, or any spatial-audio tooling can ingest directly.

Estimated build scope: 2–4 weeks of focused engineering, sequenced as: extraction first (raw input), then analysis integration (existing services slot in), then portal overlay (customers see it), then export (customers use it). None of it is research. All of it is well-understood plumbing between components that already exist.

05The Lyrai walkthrough

One phone capture. Mesh and acoustic profile. No separate rig.

The customer flow:

  1. Customer shoots a 1–3 minute phone-camera walkthrough of their space.
  2. Uploads via the Lyrai customer portal.
  3. Platform pipeline extracts video frames into a high-fidelity mesh and demuxes the spatial audio into the LiMIR analysis service in parallel.
  4. Customer portal shows a processing indicator with ETA. When done, they receive a single notification: mesh, acoustic profile, viewer link, exports — all on one Scan record, one bill, one source of truth.

This is the flow that defines the addressable market. No specialized hardware. No LiDAR phone. No dedicated acoustic mic. Any iPhone Pro user is a candidate Lyrai customer. Compared to LiDAR-only approaches the addressable market expands by roughly an order of magnitude.
