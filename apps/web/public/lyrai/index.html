01What Lyrai is

Lyrai is an acoustic-twin platform: customers capture a real-world space — a concert hall, a recording studio, a heritage site, an architectural interior — and Lyrai returns both a high-fidelity 3D model and a measurable acoustic signature of that space. The output is usable in Unity, Unreal, Wwise, FMOD, BIM tools, and standalone web viewers.

Beneath the acoustic IP — the LiMIR database (LiDAR · Material · Impulse Response), the Smart Acoustic Capture Bayesian engine, the Stony Brook research lineage — Lyrai runs as a tenant on a multi-tenant production platform that has been live for over a year, with hundreds of projects through the catalog and a working customer-facing app. The platform is the unfair advantage: most acoustic-twin proposals on the market today are 18 months of operational engineering away from being able to ship anything to a customer. Lyrai is not.

02The architecture

The customer interacts only with Lyrai. Behind it sits a platform layer that handles the heavy operational work — capture ingest, mesh processing, storage, billing, multi-tenant routing, the customer portal — so Lyrai's engineering can focus on what Lyrai uniquely does: the audio analysis, the LiMIR fingerprints, the acoustic-treatment recommendations.

The geospatial module — capture ingest, mesh post-processing, storage, viewer, exports, REST API — is what S2PX (the AEC tenant on the platform) has been running on for over a year with hundreds of customer projects through the catalog. Lyrai is the second tenant on the same module, with the audio analysis layer on top. The module is battle-tested. The Lyrai layer is what defines what Lyrai uniquely does.

Platform stack

Frontend — React 19 + Vite 6, Tailwind CSS 4, in-browser 3D viewer (Three.js), runs on phone and desktop with no plugin

Backend — Express API on Cloud Run (us-east4), 7 Gemini-powered agents, REST endpoints for the full data model

Data — Postgres via Drizzle ORM, 61 tables covering projects, scans, customers, deliverables, financials. Tenant-isolated GCS buckets for assets.

Auth + billing — Firebase Auth, multi-tier subscription support, customer portal with usage metering

Production — staging, sandbox, and production tiers. Already serving real customers.

04Spatial capabilities — already shipping

The geospatial module's customer-facing capabilities sit underneath Lyrai with no engineering investment from the Lyrai side:

  • 3D viewer in-browser — Three.js based, runs on phone or desktop, no plugin, full pan / orbit / zoom
  • Annotation overlay — pin notes to mesh coordinates, share with collaborators, tracked per project
  • Asset registry — every scan tied to a project, every project to a customer, every customer to a tenant; full audit trail
  • BIM annotation export — for AEC customers, scan deliverables route into Revit / Rhino workflows
  • Public REST API — endpoints at /api/scans, /api/scans/:id/mesh, /api/scans/:id/textures, /api/projects/:id/scans. Authenticated via tenant API keys. Lyrai's audio services consume the same API as third-party customer integrations.

06The audio layer — Lyrai's defining build

What Lyrai uniquely does sits on top of the geospatial module. Five components, all with clear interfaces to the platform underneath:

ComponentTypeWhat it does
APAC extraction serviceBuildDemuxes the 4-channel Apple Positional Audio Codec track from iPhone Pro video files. Ambisonic-style room capture with no separate mic rig.
LiMIR fingerprint integrationConnectJob queue calls LiMIR on every new scan with audio. Result stored on the Scan record as acoustic_profile.
RT60 + freq-response analysisConnectSmart Acoustic Capture's Bayesian engine consumes geometry + audio, returns structured analysis on the Scan record.
Acoustic overlay viewerBuild3D viewer extension — heat maps, propagation cones, hot spots — rendered on top of the mesh.
Spatial-audio exportBuildCustomer downloads the room's acoustic signature as an ambisonic file Unity, Wwise, or FMOD ingests directly.

Total scope: 2–4 weeks of focused engineering. Sequenced extraction → analysis → overlay → export. None of it is research. All of it is well-understood plumbing between components that already exist.

07The Lyrai walkthrough

One phone capture. Mesh and acoustic profile. No separate rig.

The customer flow:

  1. Customer shoots a 1–3 minute phone-camera walkthrough of their space.
  2. Uploads via the Lyrai customer portal.
  3. Platform pipeline extracts video frames into a high-fidelity mesh and demuxes the spatial audio into the LiMIR analysis service in parallel.
  4. Customer portal shows a processing indicator with ETA. When done, they receive a single notification: mesh, acoustic profile, viewer link, exports — all on one Scan record, one bill, one source of truth.

This is the flow that defines the addressable market. No specialized hardware. No LiDAR phone. No dedicated acoustic mic. Any iPhone Pro user is a candidate Lyrai customer. Compared to LiDAR-only approaches the addressable market expands by roughly an order of magnitude.
