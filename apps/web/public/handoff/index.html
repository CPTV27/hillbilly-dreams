Out of scope for "done building"

Things we deliberately exclude from the readiness criteria
  • Bearsville Creative summer activation — that's a new implementation, not a platform readiness gate.
  • Iconik MAM — deferred until search becomes a revenue problem.
  • LucidLink, Frame.io, Tentacle Sync — Studio C-specific tooling, gated on actual production volume justifying the spend.
  • Repo migration to measurably-better-things — separate architectural decision, not a readiness criterion.
  • Voice corpus + Pro Voice Clone — quality-of-life for Chase, not a readiness blocker.

How this charter gets used

Once a week, Cos updates the scoreboard at the top. Pass count goes up only when criteria genuinely meet the standard — no rounding up. Partial → Pass requires evidence (a test passing, a deployment confirmed, a doc landed and reviewed).

When all 24 criteria pass, Chase can move to autonomous mode. Elijah takes the operational seat. Chase becomes a Tier-A escalation target only.

If a criterion regresses (test starts failing, doc gets stale, service goes down), it goes back to Partial and the punch list reopens. The charter is a living state machine, not a one-time checklist.

Sign-off

This charter is the agreed definition of "done building." Anyone who wants to argue with it argues with the criteria, not the milestone. To change a criterion, propose a revision in writing and get Chase + Elijah on the same line.
