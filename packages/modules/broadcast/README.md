# `@mbt/modules/broadcast`

## MBT Broadcast Module (Phase C Block 4)

This module is the central orchestration hub for content distribution, repurposing, and live event management within the MBT ecosystem. It consolidates functionalities for podcast publishing, live stream management (specifically TikTok Live), and the extraction and multi-platform distribution of content clips, all while adhering to strict brand voice guidelines.

## Purpose

The primary purpose of the Broadcast Module is to streamline the complex workflow of taking raw content (live streams, recorded audio) and transforming it into ready-to-publish assets for various platforms, including podcast directories (Spotify, Apple Podcasts), video platforms (YouTube Music), and social media (Instagram Reels, TikTok). It acts as the bridge between content creation and content dissemination, ensuring consistency, efficiency, and adherence to brand-specific communication strategies.

Key responsibilities include:
*   **Podcast Publishing:** Generating RSS feeds, managing podcast shows and episodes, and distributing content to platforms like Spotify, Apple Podcasts, and YouTube Music via Buzzsprout.
*   **TikTok Live Orchestration:** Providing the data model and control mechanisms for managing TikTok Live events, including interaction with external OBS instances running on Mac mini agents.
*   **Clip Extraction Pipeline:** Facilitating the identification, extraction, enrichment, and multi-platform distribution of short-form video and audio clips from longer live broadcasts or podcast episodes.
*   **Brand Voice Routing:** Ensuring that all distributed content adheres to the distinct voice profiles associated with different brands or content verticals (e.g., "Radio" vs. "Magazine").

This module is a critical component of Phase C Block 4, enabling the automated and intelligent repurposing of high-value content across the MBT network.

## Public API

The Broadcast Module exposes a set of public functions designed to be consumed by other modules, services, or API endpoints within the MBT monorepo. These functions encapsulate the core logic for managing broadcasts, episodes, and clips.

### `scheduleBroadcast(options: ScheduleBroadcastOptions): Promise<LiveBroadcast>`

Schedules a new live broadcast event. This function creates a `LiveBroadcast` record in the database, setting up the initial state and configuration necessary for the event to go live.

*   **`options.type: 'TikTokLive' | 'YouTubeLive' | 'Twitch'`**: The platform for the live broadcast.
*   **`options.platformAccount: string`**: Identifier for the specific account on the platform (e.g., TikTok username).
*   **`options.title: string`**: Title of the broadcast.
*   **`options.description: string`**: Description of the broadcast.
*   **`options.scheduledStartTime: Date`**: Planned start time of the broadcast.
*   **`options.obsConfig: object`**: JSON object containing OBS-specific configuration (e.g., scene collection, profile, stream key template) to be used by the Mac mini agent.
*   **Returns**: A `Promise` resolving to the newly created `LiveBroadcast` object.

### `publishEpisode(options: PublishEpisodeOptions): Promise<PodcastEpisode>`

Publishes a new podcast episode to a specified podcast show. This function handles the creation of the `PodcastEpisode` record, associates it with a `PodcastShow`, generates or updates the RSS feed, and optionally triggers immediate distribution to Buzzsprout.

*   **`options.podcastShowId: string`**: The ID of the `PodcastShow` to which the episode belongs.
*   **`options.title: string`**: Title of the episode.
*   **`options.description: string`**: Detailed description of the episode.
*   **`options.audioFileUrl: string`**: URL to the hosted audio file (e.g., S3 bucket).
*   **`options.durationSeconds: number`**: Duration of the audio file in seconds.
*   **`options.releaseDate: Date`**: The date the episode should be published.
*   **`options.explicit: boolean`**: Whether the episode contains explicit content.
*   **`options.transcriptJson?: object`**: Optional JSON object containing the episode transcript.
*   **`options.autoSyncToBuzzsprout?: boolean`**: If `true`, automatically calls `syncToBuzzsprout` after episode creation. Defaults to `true`.
*   **Returns**: A `Promise` resolving to the newly created `PodcastEpisode` object.

### `syncToBuzzsprout(podcastEpisodeId: string): Promise<PodcastEpisode>`

Manually triggers or re-triggers the synchronization of a specific podcast episode to Buzzsprout. This is useful for updates or if automatic synchronization failed.

*   **`podcastEpisodeId: string`**: The ID of the `PodcastEpisode` to synchronize.
*   **Returns**: A `Promise` resolving to the updated `PodcastEpisode` object, including its `buzzsproutEpisodeId` if successful.

### `ingestClip(options: IngestClipOptions): Promise<BroadcastClip>`

Initiates the creation of a `BroadcastClip` from a segment of either a `LiveBroadcast` or a `PodcastEpisode`. This function marks the boundaries of the clip and queues it for initial processing (e.g., raw video/audio extraction).

*   **`options.sourceType: 'LiveBroadcast' | 'PodcastEpisode'`**: The type of content from which the clip is extracted.
*   **`options.sourceId: string`**: The ID of the `LiveBroadcast` or `PodcastEpisode`.
*   **`options.startTimeOffsetSeconds: number`**: Start time of the clip, relative to the source content's beginning (in seconds).
*   **`options.endTimeOffsetSeconds: number`**: End time of the clip, relative to the source content's beginning (in seconds).
*   **`options.title: string`**: A descriptive title for the clip.
*   **`options.description?: string`**: Optional description for the clip.
*   **Returns**: A `Promise` resolving to the newly created `BroadcastClip` object.

### `enrichClip(options: EnrichClipOptions): Promise<BroadcastClip>`

Applies brand-specific enrichment and processing to a `BroadcastClip`. This includes applying voice profiles, generating specific intros/outros, audio mixing, and preparing the clip for target social platforms.

*   **`options.broadcastClipId: string`**: The ID of the `BroadcastClip` to enrich.
*   **`options.voiceProfileId: string`**: The ID of the `VoiceProfile` (from the Voice System) to apply. This dictates audio characteristics, text-to-speech models, and overall brand tone.
*   **`options.targetPlatforms: ('IGReels' | 'TikTok' | 'YouTubeShorts' | 'Twitter')[ ]`**: An array of target social platforms for which the clip should be optimized (e.g., aspect ratio, duration constraints, watermark placement).
*   **`options.additionalMetadata?: object`**: Any additional platform-specific metadata or processing instructions.
*   **Returns**: A `Promise` resolving to the updated `BroadcastClip` object with processing status.

### `republishToSocial(broadcastClipId: string, targetPlatform: 'IGReels' | 'TikTok' | 'YouTubeShorts' | 'Twitter'): Promise<BroadcastClip>`

Handles the final step of pushing an enriched `BroadcastClip` to a specified social media platform. This includes API calls for upload, caption generation, hashtag application, and scheduling.

*   **`broadcastClipId: string`**: The ID of the `BroadcastClip` to publish.
*   **`targetPlatform: 'IGReels' | 'TikTok' | 'YouTubeShorts' | 'Twitter'`**: The specific social platform for publishing.
*   **Returns**: A `Promise` resolving to the updated `BroadcastClip` object, reflecting the social post's status and URL.

## Buzzsprout Integration

The Broadcast Module integrates with Buzzsprout to manage podcast distribution to major platforms like Spotify, Apple Podcasts, and YouTube Music.

*   **Functionality**: The `publishEpisode` and `syncToBuzzsprout` APIs interact directly with the Buzzsprout API. This includes uploading audio files, submitting episode metadata, and retrieving Buzzsprout-specific episode IDs.
*   **Environment Variables**: Secure API access is managed through environment variables:
    *   `BUZZSPROUT_TOKEN`: Your personal Buzzsprout API token, providing authenticated access.
    *   `BUZZSPROUT_PODCAST_ID_RADIO`: The unique Buzzsprout ID for the "Radio" brand's podcast show.
    *   `BUZZSPROUT_PODCAST_ID_MAGAZINE`: The unique Buzzsprout ID for the "Magazine" brand's podcast show (or similar per-brand IDs as needed).
*   **Mapping**: The `PodcastShow.buzzsproutPodcastId` field is used to dynamically select the correct podcast within Buzzsprout based on the `PodcastShow` record. This enables multi-show management under a single Buzzsprout account.
*   **Error Handling**: Robust error handling is implemented to catch Buzzsprout API failures, allowing for retries or manual intervention.
