## 5. Template Management

### Per-Content-Type Templates Stored in Sanity

All prompt templates are centrally managed and stored within the Sanity Content Lake. This approach offers several advantages:
*   **Flexibility**: Content teams can easily create, update, and manage templates without developer intervention.
*   **Version Control**: Sanity's inherent versioning capabilities ensure that template changes can be tracked and rolled back.
*   **Schema-Driven**: Templates are stored against a defined schema, ensuring consistency and validation.
*   **Dynamic Loading**: The `getTemplate` function dynamically fetches the most current template at runtime.

Each template document in Sanity is structured to define the main prompt, instructions for the AI model, desired output format (e.g., JSON structure, markdown headings), and placeholders for injecting resolved entities, user input, and image descriptions.

### Caching Mechanism (5min)

To optimize performance and reduce latency associated with Sanity API calls, the `getTemplate` function implements an in-memory caching layer. Templates are cached for a duration of **5 minutes**. This ensures that frequently requested templates are served quickly without repeated network requests, while still allowing for template updates to propagate within a reasonable timeframe. The cache key is composed of the `contentType`, `brandVoice`, and `organizationId` to ensure the correct template variant is retrieved.

### Brand Voice Routing (Magazine ≠ Radio enforced at template level)

A critical feature of template management is the explicit enforcement of brand voice. The `brandVoice` parameter (e.g., `'magazine'`, `'radio'`) is used as a discriminator when fetching templates from Sanity. This means:
*   A `magazine-article` template designed for a "Magazine" brand voice will be distinct from a `magazine-article` template tailored for a "Radio" brand voice.
*   These differences are baked directly into the template's instructions and example outputs, guiding the AI model to generate content that aligns perfectly with the specified tone, style, and vocabulary of the chosen brand.
*   This segregation at the template level ensures that the AI consistently produces output that respects the unique identity and communication style of different publication types or brands within the MBT ecosystem.

## 6. Entity Resolution Two-Stage Flow

The entity resolution process is designed for precision and security, involving a two-stage approach:

1.  **Search-Term Extraction via Gemini Flash**:
    *   The initial stage utilizes a powerful, efficient AI model like Google's Gemini Flash.
    *   Given a user's prompt or any textual context, Gemini Flash is employed to intelligently identify and extract potential keywords, proper nouns, and phrases that are highly likely to refer to specific business entities (e.g., company names, product names, locations, key people).
    *   This step acts as an intelligent pre-filter, transforming unstructured text into a set of focused search queries. The choice of Gemini Flash prioritizes speed and cost-effectiveness for this initial, high-volume task.

2.  **Prisma Queries with Visibility Gate**:
    *   The extracted search terms are then used to construct targeted queries against the MBT's `DirectoryBusiness` database, managed via Prisma.
    *   Crucially, every Prisma query incorporates a **visibility gate**. This gate is a security and data governance mechanism that ensures that only entities accessible or relevant to the requesting `organizationId` are returned. For example, if an organization creates content about its own products, the system will not inadvertently pull up competitor products unless explicitly configured for cross-organizational visibility.
    *   This two-stage process ensures that entity resolution is not only intelligent and accurate but also secure, respecting data access controls and organizational boundaries. It prevents "hallucinations" of entities and ensures that the AI's content is grounded in factual, accessible business data.
