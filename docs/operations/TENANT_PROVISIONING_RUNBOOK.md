# MBT Tenant Provisioning Runbook

This runbook outlines the step-by-step process for onboarding a new tenant onto the MBT platform. It covers all necessary configurations across various systems to ensure a successful and complete provisioning.

---

## 1. Prerequisites

Before initiating the provisioning process, ensure you have gathered all the necessary information and assets from the customer. This information is critical for completing the setup accurately.

*   **Business Name**: Legal and operating name of the customer's business.
    *   *Example: Vicki Wolpert, KW Athens*
*   **Desired Domain/Subdomain**: The preferred domain or subdomain for their MBT platform instance. This will be their public-facing URL.
    *   *Example: `vickiwolpert.com`*
*   **Stripe Payment Method**: Valid credit card details or bank account information for establishing the billing relationship.
    *   *Example: Credit Card details provided securely via Stripe Payment Link*
*   **Social Account Credentials/OAuth Access**:
    *   Facebook Page URL and Admin access (for Postiz connection).
    *   Instagram Business Profile URL and Admin access (for Postiz connection).
    *   LinkedIn Company Page URL and Admin access (for Postiz connection).
    *   Twitter Profile URL and Admin access (for Postiz connection).
    *   *Note: For security, direct credentials are not stored. The customer will be guided through OAuth flows within Postiz.*
*   **Brand Assets**:
    *   High-resolution Logo (SVG, PNG, or JPG).
    *   Brand Colors (Hex codes).
    *   Preferred Fonts (if custom, provide font files or Google Fonts links).
    *   Favicon (ICO or PNG).
    *   *Example: KW Athens logo, Primary Red (#DC0000), Secondary White (#FFFFFF), Fonts: Open Sans, Lato.*
*   **Existing Photos/Videos for Immich**: A link to a cloud storage (Google Drive, Dropbox) containing any media the customer wishes to pre-import.
    *   *Example: Google Drive link to "Vicki Wolpert Property Photos"*
*   **Contact Email & Phone**: Primary contact information for billing, support, and communication.
    *   *Example: vicki.wolpert@kw.com, (518) 555-1234*
*   **Desired MBT Plan**: The specific subscription tier the customer has chosen.
    *   *Example: Tuthill Pro tier*

### 2.3. Configure DNS (Cloudflare)

3.  **Configure DNS Records in Cloudflare:**
    *   **Root Domain (A Record):** Point the root domain (`@`) to Vercel's IP address.
        ```bash

        # Check Vercel documentation for the most up-to-date IPs.
        # As of writing, common Vercel IPs are 76.76.21.21, but it's best to verify.
        # Use Cloudflare UI or API.
        # Example API call (conceptual):
        # cloudflare-cli dns add --zone "$CUSTOMER_DOMAIN" --type A --name "@" --content "76.76.21.21" --ttl 1 --proxied true
        ```
        *   **UI Steps:**
            1.  Log in to Cloudflare.
            2.  Select the customer's domain (`vickiwolpert.com`).
            3.  Go to the "DNS" -> "Records" tab.
            4.  Click "Add record".
            5.  Type: `A`
            6.  Name: `@`
            7.  IPv4 address: `76.76.21.21` (Verify current Vercel IP)
            8.  TTL: `Auto`
            9.  Proxy status: `Proxied` (Orange cloud)

        # Use Cloudflare UI or API.
