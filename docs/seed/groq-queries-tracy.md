### 1. Most Recent 10 Articles

**Description:**
This query fetches the 10 most recently published articles. It's useful for quickly seeing what new content has gone live or is about to. If `publishedAt` isn't defined, it falls back to the document's creation date (`_createdAt`).

**GROQ Query:**

```groq
*[_type == "article"] | order(publishedAt desc, _createdAt desc)[0...10] {
  _id,
  title,
  "slug": slug.current,
  publishedAt,
  _createdAt,
  author->name,
  mainImage {
    asset->url,
    alt
  }
}
```

**Explanation:**
*   `*[_type == "article"]`: This is the base filter, selecting all documents that have `_type` equal to `"article"`.
*   `| order(publishedAt desc, _createdAt desc)`: This pipeline operator sorts the results. `desc` means descending, so the newest articles come first. It prioritizes `publishedAt`; if two articles have the same `publishedAt`, it then uses `_createdAt` as a secondary sort key.
*   `[0...10]`: This is a slice operator that limits the results to the first 10 documents (from index 0 up to, but not including, index 10).
*   `{ ... }`: This is a projection, shaping the output for each matching document.
    *   `_id`, `title`, `publishedAt`, `_createdAt`: These fields are directly included.
    *   `"slug": slug.current`: Renames `slug.current` to simply `slug` in the output, making it cleaner.
    *   `author->name`: Dereferences the `author` reference and fetches its `name` field.
    *   `mainImage { asset->url, alt }`: Selects the `mainImage` object and then projects specific fields from within it: the `url` from the referenced `asset` and the `alt` text.

### 2. Articles Published in the Last 30 Days

**Description:**
Finds all articles that have been published within the last 30 days from the current date. This is great for monitoring recent content output and performance.

**GROQ Query:**

```groq
*[_type == "article" && publishedAt >= dateTime(now()) - 60*60*24*30] | order(publishedAt desc) {
  _id,
  title,
  "slug": slug.current,
  publishedAt,
  author->name,
  status
}
```

**Explanation:**
*   `*[_type == "article"]`: Selects all documents of type `article`.
*   `&& publishedAt >= dateTime(now()) - 60*60*24*30`: This is the core filter for the date range.
    *   `dateTime(now())`: Gets the current date and time as a `dateTime` object.
    *   `60*60*24*30`: Calculates the number of milliseconds in 30 days (60 seconds * 60 minutes * 24 hours * 30 days).
    *   By subtracting this from `dateTime(now())`, we get a timestamp representing 30 days ago. The query then filters for articles where `publishedAt` is greater than or equal to this timestamp.
*   `| order(publishedAt desc)`: Sorts the results by `publishedAt` in descending order, showing the newest first.
*   `{ ... }`: Projects the `_id`, `title`, `slug.current` (renamed to `slug`), `publishedAt`, `author`'s `name`, and `status` of the article.

### 4. Upcoming Events in the Next 30 Days

**Description:**
Retrieves events that are scheduled to occur from today up to the next 30 days. Useful for event management and promotions.

**GROQ Query:**

```groq
*[_type == "event" && dateTime(date) >= dateTime(now()) && dateTime(date) < dateTime(now()) + 60*60*24*30] | order(date asc) {
  _id,
  title,
  date,
  location,
  "isUpcoming": dateTime(date) >= dateTime(now()), // Custom boolean field
  "daysUntilEvent": ceil((dateTime(date) - dateTime(now())) / (1000 * 60 * 60 * 24))
}
```

**Explanation:**
*   `*[_type == "event"]`: Selects all documents of type `event`.
*   `&& dateTime(date) >= dateTime(now())`: Filters for events where the `date` field is today or in the future.
*   `&& dateTime(date) < dateTime(now()) + 60*60*24*30`: Further filters for events occurring within the next 30 days. The `+ 60*60*24*30` adds 30 days in milliseconds to the current time.
*   `| order(date asc)`: Sorts the results by `date` in ascending order, so the soonest events appear first.
*   `{ ... }`: Projects `_id`, `title`, `date`, and `location`.
    *   `"isUpcoming": dateTime(date) >= dateTime(now())`: A custom projected field that returns `true` if the event is in the future, `false` otherwise.
    *   `"daysUntilEvent": ceil((dateTime(date) - dateTime(now())) / (1000 * 60 * 60 * 24))`: Calculates the number of full days until the event. `ceil()` rounds up to the nearest whole number.

### 5. Draft Articles Waiting for Review

**Description:**
Finds all articles that are currently in a draft state and marked with a `status` of "pendingReview". This is essential for editorial workflows to track content that needs attention before publication.

**GROQ Query:**

```groq
*[_type == "article" && _id in path("drafts.**") && status == "pendingReview"] | order(_updatedAt desc) {
  _id,
  title,
  "slug": slug.current,
  author->name,
  status,
  _updatedAt,
  _createdAt
}
```

**Explanation:**
*   `*[_type == "article"]`: Selects all documents of type `article`.
*   `&& _id in path("drafts.**")`: This is the key to identifying draft documents. Sanity automatically prefixes draft document `_id`s with `drafts.`. `path("drafts.**")` creates a path matcher for any ID starting with `drafts.`.
*   `&& status == "pendingReview"`: Filters for articles where the `status` field (assuming you have one) is explicitly set to `"pendingReview"`.
*   `| order(_updatedAt desc)`: Sorts the results by the last update time in descending order, showing the most recently modified drafts first.
*   `{ ... }`: Projects `_id`, `title`, `slug.current` (renamed to `slug`), `author`'s `name`, `status`, `_updatedAt`, and `_createdAt`.
