## KEY DESIGN PRINCIPLES FROM THE BENCHMARKS

1. **Linear's lesson:** Micro-animations and keyboard shortcuts make software feel alive. Every interaction should have tactile feedback.
2. **Stripe's lesson:** Every surface is polished — landing pages, dashboards, docs, error states. No "B-tier" pages.
3. **Airbnb's lesson:** Big photography + card layouts = instant trust. The directory needs this energy.
4. **Spotify's lesson:** Dark UI with one accent color is iconic. Entertainment division should own this.
5. **Duolingo's lesson:** Streaks and encouragement drive daily usage. The morning briefing should feel like a streak.
6. **Uber's lesson:** Three taps to completion. The decision queue should be this ruthlessly simple.
7. **Notion's lesson:** Whitespace is confidence. Don't fill every pixel.

## THE ANTI-PATTERNS (what NOT to do)

- Complex dashboards with 12 widgets (the thing we're replacing)
- Hamburger menus hiding essential features
- Settings pages with 40 toggles
- Onboarding wizards with 7 steps
- "Dashboard" as the default view
- Feature-comparison tables as the hero (save for pricing page only)
