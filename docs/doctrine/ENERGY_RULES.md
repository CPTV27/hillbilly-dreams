# Founder Energy Rules

**Status:** Operational doctrine governing Chase's time + cognitive resources.
**Last revised:** 2026-05-12

The scarcest resource in the Big Muddy ecosystem is not money. It is **Chase's cognition.**

Any agent (Cos, Patch, an LLM, a contractor, Chase himself) proposing work that consumes Chase's time is implicitly drawing down this scarce resource. The rules below apply.

---

## Core principles

1. **Chase cognition is the scarcest resource in the ecosystem.** More scarce than money. More scarce than partner time. More scarce than calendar slots.
2. **Any recurring maintenance burden must justify itself economically.** A weekly task that takes 2 hours is ~100 hours/year of Chase's most expensive resource. That must produce ~$5,000/year in direct value at minimum — or it should not exist.
3. **Platform work competes directly against photography revenue.** Every hour Chase spends on infrastructure maintenance is an hour not on photography client work. CPP generates ~$100k/yr; that's ~$50/hr opportunity cost minimum. Platform work that doesn't produce that level of value is a net negative.
4. **Simpler systems are preferred unless complexity compounds authority.** Custom code is justified only when it produces a defensible asset (e.g., the Directory). Custom code that exists because "we built it once" is sunk cost.
5. **Reduce context-switching whenever possible.** Batch decisions. Batch communications. Batch infrastructure work into clear sprints rather than scattered maintenance.

---

## Time hierarchy

Chase's time goes to these categories in priority order:

| Priority | Category | Why |
|---|---|---|
| 1 | Photography client work | Revenue. Funds the ecosystem. |
| 2 | Direct BMT growth activities — destination events, sponsor outreach, artist relationships | The growth bet. Compounds. |
| 3 | Partner coordination — Tracy + Amy + Sean + Studio C + Tuthill | Maintains the human infrastructure. |
| 4 | Magazine + content production — the identity engine | Compounds cultural authority. |
| 5 | Infrastructure & platform maintenance | Pure cost. Minimize. |
| 6 | Aspirational / speculative work (Bearsville, MBT productization, etc.) | Deferred to Y2+ per doctrine. |

If two activities compete for Chase's time, the higher-priority one wins. Default.

---

## Hard rules

- **Photography work gets scheduling priority.** Block photography client time first, then schedule BMT around it. Not the other way around.
- **Platform work is allocated in sprints, not continuous trickle.** No "I'll just spend an hour fixing this" without a sprint context. Sprints batch the context-switching cost.
- **A recurring task that consumes 2+ hrs/wk must have a written justification.** Document why it exists and what would let us kill it.
- **Operational tasks that can be off-the-shelf-ed for <$50/mo should be off-the-shelf-ed.** Chase's hour is worth more than the SaaS subscription. The exception is when the off-the-shelf option fails the "could Tracy or Amy use this without help" test.
- **Partner coordination time is non-negotiable.** Cutting partner conversation time to save Chase's hours is false economy. Partner misalignment costs more than the time saved.

---

## Anti-patterns to flag

When proposing work, watch for these patterns and reject them:

1. **"It'll only take an hour."** Underestimates context-switching cost. Real cost is often 2–3x.
2. **"We could automate this."** Automation has its own maintenance burden. Compute the lifetime cost, not the one-time cost.
3. **"This will be useful eventually."** Optionality has a cost. Speculative work is the largest source of recurring drag.
4. **"It's a quick fix."** Fixes compound into systems that need maintenance. Prefer not-having-the-thing over fixing-the-thing.
5. **"We need to be ready for X."** Readiness for unconfirmed futures is one of the most expensive things a small team can pay for.

---

## How agents should apply this

Before proposing any work that involves Chase's time:

1. **Estimate the hours per week** the work will consume, including ongoing maintenance.
2. **Calculate the lifetime hour cost** (weekly hours × 52 weeks × however many years it persists).
3. **State the expected direct economic value** the work produces.
4. **State the alternative use of Chase's time** if this work weren't done.
5. **If the economic value doesn't beat the alternative, redesign or kill.**

A proposal that bypasses this calculation is implicitly asking Chase to subsidize the work with personal cognition. Don't do that without acknowledging it explicitly.
