## `@bigmuddy/commerce` API Reference

The `@bigmuddy/commerce` module provides a robust set of functionalities to integrate e-commerce and subscription management into your application. It acts as an abstraction layer over underlying payment gateways and database operations, offering a simplified interface for common commerce tasks.

### Import Patterns

The module exports several top-level objects, each containing related functions. You can import them individually or as a single object:

```typescript
// Import specific modules
import { plans, subscriptions, products, orders, checkout, webhooks } from '@bigmuddy/commerce';

// Import all modules into a single namespace
import * as commerce from '@bigmuddy/commerce';

// Import types
import { Plan, Subscription, Product, Order, PlanInterval, OrderStatus } from '@bigmuddy/commerce';
```

### Environment Variable Requirements

The `@bigmuddy/commerce` module relies on several environment variables for configuration, particularly for integrating with payment gateways like Stripe and for secure webhook processing. Ensure these are set in your deployment environment:

*   `STRIPE_SECRET_KEY`: Your Stripe secret API key (e.g., `sk_live_...` or `sk_test_...`). Required for most Stripe-related operations.
*   `STRIPE_WEBHOOK_SECRET`: The secret key for verifying Stripe webhooks. Obtain this from your Stripe dashboard under Webhooks settings.
*   `BIGMUDDY_DB_CONNECTION_STRING`: The connection string for your database, where commerce data (plans, products, orders, subscriptions) is stored. (e.g., PostgreSQL, MongoDB connection string).
*   `BIGMUDDY_FRONTEND_URL`: The base URL of your frontend application. Used for redirecting users after checkout or customer portal sessions.

### Common Usage Workflow

A typical integration workflow with `@bigmuddy/commerce` might look like this:

1.  **Define Plans and Products**:
    *   Use `plans.create` to define different subscription tiers (e.g., Basic, Premium).
    *   Use `products.create` to define physical or digital goods.
    *   Optionally, use `plans.setStripeIds` to link your internal plans to Stripe Price IDs after creation in Stripe.
2.  **Display Options**:
    *   Fetch available plans using `plans.list` or `plans.get` to display subscription options to users.
    *   Fetch available products using `products.list` or `products.get` for your storefront.
3.  **Customer Checkout**:
    *   For new subscriptions, use `checkout.createSubscriptionCheckout` to generate a Stripe Checkout session URL.
    *   For one-time product purchases, use `checkout.createOrderCheckout` to generate a Stripe Checkout session URL.
    *   Redirect your user to the generated URL.
4.  **Handle Webhooks**:
    *   Set up a webhook endpoint in your application to receive events from Stripe (e.g., `checkout.session.completed`, `invoice.payment_succeeded`, `customer.subscription.deleted`).
    *   Use `webhooks.verifyAndParse` to validate the incoming webhook event.
    *   Use `webhooks.handle` to process the event, which will update the internal state of subscriptions, orders, and other commerce entities in your database. This is crucial for keeping your system in sync with Stripe.
5.  **Manage Subscriptions/Orders**:
    *   Allow users to manage their subscriptions via a customer portal created with `checkout.createCustomerPortal`.
    *   Programmatically manage subscriptions using functions like `subscriptions.cancelAtPeriodEnd`, `subscriptions.cancelImmediately`, `subscriptions.updateStatus`, or `subscriptions.bulkChangePlan`.
    *   Process orders using `orders.markPaid`, `orders.markFulfilled`, `orders.markShipped`, and `orders.refund`.
6.  **Reporting**:
    *   Retrieve subscription data using `subscriptions.list` or `subscriptions.get`.
    *   Calculate metrics like Monthly Recurring Revenue (MRR) using `subscriptions.getMRR`.
    *   Access order history with `orders.list`.

## Types

### Type: `TenantId`

`type TenantId = string;`

Description: A unique identifier for a tenant (e.g., an organization or workspace) within your application. All commerce operations are scoped to a specific tenant.

### Type: `BrandId`

`type BrandId = string;`
