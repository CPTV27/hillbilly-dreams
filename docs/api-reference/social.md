# `@bigmuddy/social` API Reference

The `@bigmuddy/social` module provides a robust set of tools for managing and publishing social media content across various platforms while ensuring strict brand voice consistency and preventing cross-brand contamination. It includes functionalities for resolving brand-specific voice profiles, validating content against these profiles, and orchestrating the publishing process.

---

## Types

### `Platform`

Represents the social media platform to which content will be published. This is a union of string literals, ensuring type safety and limiting platform options.

```typescript
type Platform = 'Twitter' | 'Facebook' | 'LinkedIn' | 'Instagram' | 'TikTok';
```

**Members:**

*   `'Twitter'`: Content intended for Twitter.
*   `'Facebook'`: Content intended for Facebook.
*   `'LinkedIn'`: Content intended for LinkedIn.
*   `'Instagram'`: Content intended for Instagram.
*   `'TikTok'`: Content intended for TikTok.

### `Brand`

Defines a specific brand entity within the Big Muddy ecosystem. Each brand is associated with a unique voice profile and may have specific restrictions regarding its compatibility with other brands or voice profiles.

```typescript
interface Brand {
  /**
   * A unique identifier for the brand.
   * @example 'BM_Corp'
   */
  id: string;

   */
```

### `VoiceProfile`

Describes the distinct voice and tone guidelines for a brand. This includes stylistic elements, preferred keywords, and phrases that are strictly forbidden. Voice profiles are central to maintaining brand identity and consistency.

```typescript
interface VoiceProfile {
```

## Functions

### `voiceRouter`

The `voiceRouter` object provides utilities for resolving and validating brand voice profiles. It is critical for ensuring content adheres to established brand guidelines and preventing cross-brand contamination.

#### `voiceRouter.resolveVoiceProfile(brandId: string): VoiceProfile`

Resolves and returns the `VoiceProfile` associated with a given `brandId`. This function is the primary mechanism for retrieving the voice guidelines that apply to content intended for a specific brand.

**Parameters:**

*   `brandId`: `string`
    *   The unique identifier of the brand for which to retrieve the voice profile.

**Returns:**

*   `VoiceProfile`
    *   The `VoiceProfile` object corresponding to the provided `brandId`.

**Throws:**

*   `Error`: If no brand is found for the given `brandId`.
*   `Error`: If the brand found does not have an associated `voiceProfileId`.
*   `Error`: If the `voiceProfileId` associated with the brand does not resolve to an existing `VoiceProfile`.

**Example:**

```typescript
import { voiceRouter } from '@bigmuddy/social';

try {
  const corporateVoice = voiceRouter.resolveVoiceProfile('BM_Corp');
  console.log(`Corporate Voice Profile: ${corporateVoice.name}`);
  // Expected output: Corporate Voice Profile: Corporate Professional
} catch (error) {
  console.error(`Failed to resolve voice profile: ${error.message}`);
}
```

#### `voiceRouter.assertCompatible(brandId: string, voiceProfileId: string): void`

Performs a strict compatibility check between a given `brandId` and a `voiceProfileId`. This function is a **hard-fail guard** designed to prevent cross-brand contamination. It checks both the brand's `incompatibleWith` list and the voice profile's `incompatibleWith` list to ensure no explicit conflicts exist.

**Parameters:**

*   `brandId`: `string`
    *   The ID of the brand to check for compatibility.
*   `voiceProfileId`: `string`
    *   The ID of the voice profile to check against the brand.

**Returns:**

*   `void`
    *   If no incompatibility is found, the function completes without returning a value.

**Throws:**

*   `Error`: If the specified `brandId` is not found.
*   `Error`: If the specified `voiceProfileId` is not found.
*   `Error`: If the `brandId` is explicitly listed in the `incompatibleWith` array of the `voiceProfileId`.
*   `Error`: If the `voiceProfileId` is explicitly listed in the `incompatibleWith` array of the `brandId`.
*   `Error`: If the `voiceProfileId` is the *default* voice profile for a brand, but that brand's `incompatibleWith` list contains the `brandId` itself (a self-referential guard against misconfiguration).

**Example:**

```typescript
import { voiceRouter } from '@bigmuddy/social';

try {
  // This should pass if BM_Corp and VP_CorporateProfessional are compatible
  voiceRouter.assertCompatible('BM_Corp', 'VP_CorporateProfessional');
  console.log('BM_Corp and Corporate Professional voice profile are compatible.');

  // This might throw an error if BM_Corp is incompatible with VP_YouthfulPlayful
  voiceRouter.assertCompatible('BM_Corp', 'VP_YouthfulPlayful'); // Assuming this throws
  console.log('This line will not be reached if an error is thrown.');
} catch (error) {
  console.error(`Compatibility check failed: ${error.message}`);
  // Expected output (if incompatible): Compatibility check failed: Brand 'BM_Corp' is incompatible with voice profile 'VP_YouthfulPlayful'.
}
```

#### `voiceRouter.findForbidden(content: string, voiceProfileId: string): string[]`

Analyzes a given content string against the `forbiddenPhrases` defined in a specified `VoiceProfile`. This function helps ensure that no prohibited language is used in brand communications.

**Parameters:**

*   `content`: `string`
    *   The text content to scan for forbidden phrases.
*   `voiceProfileId`: `string`
    *   The ID of the `VoiceProfile` whose `forbiddenPhrases` should be used for validation.

**Returns:**

*   `string[]`
    *   An array of strings, where each string is a forbidden phrase found within the `content`.
    *   If no forbidden phrases are found, an empty array is returned.

**Throws:**

*   `Error`: If the specified `voiceProfileId` is not found.

**Example:**

```typescript
import { voiceRouter } from '@bigmuddy/social';

const edgyContent = 'Our new product is totally lit and will make you YOLO!';
const professionalVoiceId = 'VP_CorporateProfessional';

try {
  const forbidden = voiceRouter.findForbidden(edgyContent, professionalVoiceId);

  if (forbidden.length > 0) {
    console.warn(`Content contains forbidden phrases for ${professionalVoiceId}: ${forbidden.join(', ')}`);
    // Expected output: Content contains forbidden phrases for VP_CorporateProfessional: lit, YOLO
  } else {
    console.log('Content is clean.');
  }
} catch (error) {
  console.error(`Failed to check forbidden phrases: ${error.message}`);
}
```

### `publisher`

The `publisher` object encapsulates the core logic for publishing social media content, integrating brand voice validation and compatibility checks.

#### `publisher.publish(input: PublishInput): Promise<PublishResult>`

Initiates the process of publishing social media content. This asynchronous function performs several critical steps:
1.  Resolves the `VoiceProfile` for the given `brandId`.
2.  Performs a strict compatibility check using `voiceRouter.assertCompatible`.
3.  Scans the content for `forbiddenPhrases` using `voiceRouter.findForbidden`.
4.  If all checks pass, it simulates (or dispatches to an actual) platform-specific publishing mechanism.

**Parameters:**

*   `input`: `PublishInput`
    *   An object containing all necessary details for the publication, including `brandId`, `platform`, `content`, and optional scheduling information.

**Returns:**

*   `Promise<PublishResult>`
    *   A promise that resolves to a `PublishResult` object, indicating the success or failure of the publication attempt.

**Throws:**

*   This function does not directly throw synchronous errors; instead, it returns a `PublishResult` with `success: false` and an `error` message in case of validation failures or internal publishing issues.

**Example:**

```typescript
import { publisher, Platform } from '@bigmuddy/social';

async function publishMyContent() {
  const input = {
    brandId: 'BM_Corp',
    platform: 'LinkedIn' as Platform,
    content: 'We are excited to announce our Q3 strategic growth initiatives. Learn more on our website!',
    targetAudience: 'Investors and Business Partners',
  };

  try {
    const result = await publisher.publish(input);

    if (result.success) {
      console.log(`Content published successfully! Post ID: ${result.postId}`);
    } else {
      console.error(`Failed to publish content: ${result.error}`);
    }
  } catch (err) {
    console.error(`An unexpected error occurred during publishing: ${err.message}`);
  }

  // Example of a failing publication due to forbidden phrases
  const badInput = {
    brandId: 'BM_Corp',
    platform: 'Twitter' as Platform,
    content: 'Our new corporate strategy is totally lit and will totally make us YOLO to success!',
  };

  const badResult = await publisher.publish(badInput);
  if (!badResult.success) {
    console.error(`Failed to publish bad content as expected: ${badResult.error}`);
    // Expected output: Failed to publish bad content as expected: Content rejected: Forbidden phrases detected: lit, YOLO
  }
}

publishMyContent();
```

## Cross-brand Contamination Guards

The `@bigmuddy/social` module places a paramount emphasis on maintaining strict brand integrity and preventing any inadvertent cross-brand contamination. This is crucial for legal compliance, preserving brand reputation, and avoiding confusing customers or diluting brand messaging. Two primary mechanisms enforce these guards: the `assertCompatible()` hard-fail and the per-brand/voice profile `incompatibleWith` arrays.

### The Purpose of Contamination Guards

In large organizations like Big Muddy with multiple distinct brands, sub-brands, or even competing product lines, it's critical to ensure that:

1.  **Brand Voice Consistency:** Content published for Brand A strictly adheres to Brand A's voice profile and never inadvertently adopts elements of Brand B's voice.
2.  **Reputation Protection:** A controversial or edgy voice profile designed for one brand does not accidentally get applied to a more conservative corporate brand, leading to PR disasters.
3.  **Legal and Regulatory Compliance:** Certain brands may operate under different legal frameworks or have specific restrictions (e.g., healthcare vs. entertainment). Mixing these can have severe consequences.
4.  **Market Differentiation:** Each brand needs to maintain its unique identity in the market. Mixing messaging can blur lines and confuse target audiences.

The contamination guards are designed to catch potential issues *before* content ever reaches a public platform.

### `assertCompatible()`: The Hard-Fail Mechanism

The `voiceRouter.assertCompatible(brandId, voiceProfileId)` function is the cornerstone of the cross-brand contamination guards. Its design philosophy is **"fail fast and loudly."**

When `assertCompatible()` is called, it performs a bidirectional check:

1.  **Brand-to-Voice Compatibility:** It checks if the provided `voiceProfileId` is listed in the `incompatibleWith` array of the `Brand` specified by `brandId`.
2.  **Voice-to-Brand Compatibility:** It checks if the provided `brandId` is listed in the `incompatibleWith` array of the `VoiceProfile` specified by `voiceProfileId`.

If *any* incompatibility is detected in either direction, `assertCompatible()` **immediately throws an `Error`**. This is not a warning; it is a critical failure that prevents the operation from proceeding. This hard-fail approach ensures that:

*   **No Ambiguity:** There's no scenario where potentially incompatible content is allowed to slip through with just a warning.
*   **Developer Awareness:** Developers are forced to address compatibility issues directly and cannot bypass them inadvertently.
*   **System Integrity:** The publishing pipeline cannot continue if a fundamental brand integrity rule is violated.

This function is implicitly called by `publisher.publish` to ensure that any content being prepared for publication is first validated against these strict compatibility rules.

### Per-Brand and Per-Voice Profile `incompatibleWith` Arrays

Both the `Brand` and `VoiceProfile` types include an `incompatibleWith: string[]` array. These arrays are the declarative definitions of incompatibility rules.

*   **`Brand.incompatibleWith`**: This array lists the IDs of other `Brand` entities or `VoiceProfile` entities that this specific brand is *not allowed* to be associated with. For example, a "Corporate" brand might list an "Edgy Startup" brand's ID or its "Youthful Playful" voice profile ID here.
*   **`VoiceProfile.incompatibleWith`**: This array lists the IDs of `Brand` entities or other `VoiceProfile` entities that *this specific voice profile* is not allowed to be used with. For example, a "Youthful Playful" voice profile might list a "Corporate" brand's ID or a "Formal Legal" voice profile ID here.

The dual nature of these arrays provides comprehensive coverage:

*   A brand can explicitly forbid certain voices from being used with it.
*   A voice profile can explicitly forbid itself from being used by certain brands.

This allows for a flexible yet robust matrix of compatibility rules, preventing scenarios where an incompatibility might only be declared on one side.

**Example Scenario:**

Consider two brands:
*   `BM_Corp` (Big Muddy Corporate) - formal, professional.
*   `BM_Gaming` (Big Muddy Gaming Division) - casual, edgy, playful.

And two voice profiles:
*   `VP_CorporateProfessional`
*   `VP_YouthfulPlayful`

To prevent `BM_Corp` from ever using `VP_YouthfulPlayful`, and vice-versa, the `incompatibleWith` arrays would be configured as follows:

```typescript
// --- Brand Definitions ---
const BRANDS: { [id: string]: Brand } = {
  'BM_Corp': {
    id: 'BM_Corp',
    name: 'Big Muddy Corporation',
    voiceProfileId: 'VP_CorporateProfessional',
    incompatibleWith: ['BM_Gaming', 'VP_YouthfulPlayful'], // BM_Corp cannot be associated with BM_Gaming or its youthful voice
  },
  'BM_Gaming': {
    id: 'BM_Gaming',
    name: 'Big Muddy Gaming',
    voiceProfileId: 'VP_YouthfulPlayful',
    incompatibleWith: ['BM_Corp', 'VP_CorporateProfessional'], // BM_Gaming cannot be associated with BM_Corp or its formal voice
  },
  // ... other brands
};

// --- VoiceProfile Definitions ---
const VOICE_PROFILES: { [id: string]: VoiceProfile } = {
  'VP_CorporateProfessional': {
    id: 'VP_CorporateProfessional',
    name: 'Corporate Professional',
    description: 'Authoritative, informative, and formal, suitable for corporate announcements.',
    tone: 'Formal',
    keywords: ['innovation', 'solution', 'partnership', 'strategic growth', 'stakeholders'],
    forbiddenPhrases: ['cringe', 'lit', 'YOLO', 'bet', 'bro', 'epic fail'],
    incompatibleWith: ['BM_Gaming', 'VP_YouthfulPlayful'], // This voice cannot be used for BM_Gaming or with the youthful voice
  },
  'VP_YouthfulPlayful': {
    id: 'VP_YouthfulPlayful',
    name: 'Youthful and Playful',
    description: 'Energetic, informal, and engaging, targeting a younger demographic.',
    tone: 'Playful',
    keywords: ['game', 'level up', 'community', 'fun', 'challenge', 'hype'],
    forbiddenPhrases: ['synergy', 'leverage', 'paradigm', 'Q3 earnings', 'shareholder value'],
    incompatibleWith: ['BM_Corp', 'VP_CorporateProfessional', 'VP_LegalCompliance'], // This voice cannot be used for BM_Corp or with formal/legal voices
  },
  'VP_LegalCompliance': {
    id: 'VP_LegalCompliance',
    name: 'Legal and Compliance',
    description: 'Precise, factual, and cautious, used for legal disclaimers and regulatory updates.',
    tone: 'Strictly Factual',
    keywords: ['regulatory', 'compliance', 'terms', 'conditions', 'disclaimer', 'liability'],
    forbiddenPhrases: ['awesome', 'amazing', 'fun', 'exciting', 'easy', 'guarantee'],
    incompatibleWith: ['BM_Gaming', 'VP_YouthfulPlayful', 'VP_MarketingCreative'], // Legal voice should not mix with playful or creative marketing
  },
  'VP_MarketingCreative': {
    id: 'VP_MarketingCreative',
    name: 'Marketing Creative',
    description: 'Engaging, persuasive, and benefit-driven, focused on product promotion.',
    tone: 'Persuasive',
    keywords: ['discover', 'explore', 'transform', 'experience', 'unleash', 'benefit'],
    forbiddenPhrases: ['liability', 'regulatory', 'compliance', 'terms and conditions'],
    incompatibleWith: ['VP_LegalCompliance'], // Marketing voice should not overlap with legal strictness
  },
  // ... other voice profiles
};
```

In this setup, if `publisher.publish` receives an `input` with `brandId: 'BM_Corp'` and tries to publish content that implicitly or explicitly references `VP_YouthfulPlayful` (e.g., if a developer accidentally specified `VP_YouthfulPlayful` in a custom publishing flow, or if `assertCompatible` was used directly), `assertCompatible()` would immediately throw an error, preventing the publication.

This robust system of explicit incompatibilities and hard-fail checks ensures that Big Muddy's diverse portfolio of brands maintains its distinct identity and integrity across all social media communications.
