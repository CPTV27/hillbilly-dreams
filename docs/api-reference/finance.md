## 1. Core Concepts

The `@bigmuddy/finance` module revolves around several core concepts:

*   **Engagement:** Represents a formal agreement or project with a client. It tracks the status, start/end dates, and associated financial terms.
*   **Bucket:** Within an engagement, hours or resources are often allocated to specific "buckets" (e.g., "Development Hours," "Support Hours," "Retainer"). This module allows logging hours against these buckets.
*   **MRR (Monthly Recurring Revenue):** A key financial metric tracking predictable, recurring revenue generated from active engagements and subscriptions.
*   **PnL (Profit & Loss):** A financial statement summarizing the revenues, costs, and expenses incurred during a specific period, providing insight into an entity's profitability.
*   **Platform Fee:** A recurring charge associated with the use of the platform, often calculated based on engagement terms.

### `EntityId`

### `EngagementStatus`

Represents the current lifecycle stage of an engagement.

```typescript
type EngagementStatus = 'active' | 'paused' | 'ended' | 'pending';
```

**Possible Values:**

*   `'active'`: The engagement is currently ongoing and billable.
*   `'paused'`: The engagement is temporarily on hold; no new hours can be logged, and recurring charges might be suspended.
*   `'ended'`: The engagement has concluded.
*   `'pending'`: The engagement has been created but not yet started (e.g., awaiting client approval or a future start date).

### `ModuleLicenseProfile`

Defines the licensing terms for a specific module or feature within an engagement.

```typescript
interface ModuleLicenseProfile {
    moduleId: EntityId;
    name: string;
    licenseType: 'per-user' | 'flat-fee' | 'tiered';
    baseFee: number; // Base fee for the license
    unitCost?: number; // Cost per user or per tier unit
    maxUsers?: number; // Maximum users for 'per-user' or a tier boundary
    minUsers?: number; // Minimum users for 'per-user' or a tier boundary
    billingCycle: 'monthly' | 'annually';
    activeUsers?: number; // Current number of active users for 'per-user' licenses
}
```

**Properties:**

*   `moduleId`: `EntityId` - The unique identifier for the licensed module.
*   `name`: `string` - The human-readable name of the module (e.g., "CRM Pro," "Analytics Suite").
*   `licenseType`: `'per-user' | 'flat-fee' | 'tiered'` - The pricing model for the license.
    *   `'per-user'`: Billed based on the number of active users.
    *   `'flat-fee'`: A fixed fee, regardless of usage.
    *   `'tiered'`: Pricing changes based on usage tiers (e.g., 1-10 users, 11-50 users).
*   `baseFee`: `number` - The foundational cost for the license, applied before any unit or tier-specific charges.
*   `unitCost`: `number` (Optional) - The cost per unit (e.g., per user) if `licenseType` is `'per-user'` or part of a tiered model.
*   `maxUsers`: `number` (Optional) - The upper limit of users for this license (relevant for `per-user` or tiered models).
*   `minUsers`: `number` (Optional) - The lower limit of users for this license (relevant for `per-user` or tiered models).
*   `billingCycle`: `'monthly' | 'annually'` - How frequently the license is billed.
*   `activeUsers`: `number` (Optional) - The current count of active users for this module, particularly relevant for `per-user` licenses.
