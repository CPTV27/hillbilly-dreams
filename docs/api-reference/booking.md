## Module Overview

The `@bigmuddy/booking` module provides a programmatic interface to manage the core entities of a booking system. It's designed to be used in server-side applications or secure client-side environments with appropriate authentication. The module exposes a set of asynchronous functions categorized by the primary entities they interact with: Resources, Bookings, Tickets, and Quotes.

To get started, import the module:

```typescript
import * as booking from '@bigmuddy/booking';
// Or for specific functions:
import { getBooking, createResource } from '@bigmuddy/booking';
```

## Authentication and Authorization

All API calls to the `@bigmuddy/booking` module require proper authentication and authorization. While the specifics of token acquisition and management are outside the scope of this reference, it's assumed that an authenticated context (e.g., an API key, JWT, or OAuth token) is implicitly handled by the underlying infrastructure or passed via a global configuration. ## Common Patterns

### List Options

Many `list` methods accept an optional `ListOptions` object for pagination, filtering, and sorting.

```typescript
interface ListOptions {
  limit?: number; // Maximum number of items to return (default: 20, max: 100)
  offset?: number; // Number of items to skip (for pagination)
  sortBy?: string; // Field to sort by (e.g., 'createdAt', 'name')
  sortOrder?: 'asc' | 'desc'; // Sort order (ascending or descending)
  filter?: {
    [key: string]: string | number | boolean | string[]; // Key-value pairs for filtering (e.g., { status: 'CONFIRMED' })
  };
}
```

### Error Handling

API calls are asynchronous and return Promises. Errors are typically instances of custom error classes extending `Error`, providing more context.

Common error types include:

*   `NotFoundError`: The requested resource could not be found (HTTP 404).
*   `ValidationError`: Input parameters failed validation (HTTP 400).
*   `ConflictError`: The operation conflicts with the current state of the resource (HTTP 409).
*   `AuthenticationError`: No valid authentication credentials provided (HTTP 401).
*   `AuthorizationError`: Authenticated user lacks necessary permissions (HTTP 403).
*   `ServiceUnavailableError`: The service is temporarily unavailable (HTTP 503).
*   `InternalServerError`: An unexpected error occurred on the server (HTTP 500).

Example error handling:

```typescript
try {
  const booking = await booking.getBooking('some-invalid-id');
} catch (error) {
  if (error instanceof booking.NotFoundError) {
    console.error('Booking not found:', error.message);
  } else if (error instanceof booking.ValidationError) {
    console.error('Invalid input:', error.message);
  } else {
    console.error('An unexpected error occurred:', error);
  }
}
```

### `reserveResource`

Manually reserves a specific resource for a given time period, associating it with a booking. This is typically used for direct resource allocation outside the standard booking flow or for internal system adjustments.

```typescript
async reserveResource(
  resourceId: string,
  bookingId: string,
  startTime: Date,
  endTime: Date
): Promise<Resource>;
```

*   **Description**: Explicitly reserves a resource for a specified `bookingId` and time range. This operation checks for existing conflicts.
*   **Parameters**:
    *   `resourceId` (required) - `string`: The ID of the resource to reserve.
    *   `bookingId` (required) - `string`: The ID of the booking this reservation is for.
    *   `startTime` (required) - `Date`: The start time of the reservation.
    *   `endTime` (required) - `Date`: The end time of the reservation.
*   **Returns**: `Promise<Resource>` - A promise that resolves to the updated `Resource` object.
*   **Possible Errors**: `NotFoundError` (resource or booking not found), `ConflictError` (resource already reserved for the given time), `ValidationError` (invalid time range), `AuthenticationError`, `AuthorizationError`, `InternalServerError`.
*   **Example**:
    ```typescript
    const now = new Date();
    const oneHourLater = new Date(now.getTime() + 60 * 60 * 1000);
    const updatedResource = await booking.reserveResource(
      'res_abc123',
      'bkg_xyz789',
      now,
      oneHourLater
    );
    console.log(`Resource ${updatedResource.name} reserved for booking bkg_xyz789.`);
    ```
