## 1. Core Concepts and Types

This section defines the fundamental data structures used throughout the `@bigmuddy/broadcast` module. These types represent the entities managed by the system, such as podcasts, episodes, live streams, and clips, along with their associated metadata and status.

---

### PodcastShow

Represents a single podcast series or show.

```typescript
interface PodcastShow {
  id: string;
  brandId: string;
  name: string;
  slug: string; // Unique URL-friendly identifier for the show
  description?: string;
  imageUrl?: string; // URL to the show's cover art
  authorName?: string;
  language?: string; // e.g., 'en-us'
  feedUrl?: string; // The URL where the RSS feed for this show is published
```


### BroadcastClip

```typescript
interface BroadcastClip {
```

### BroadcastStatus

An enumeration of possible states for a `LiveBroadcast`.

```typescript
type BroadcastStatus =
  | 'SCHEDULED' // The broadcast is planned but not yet live.
  | 'LIVE'      // The broadcast is currently active.
  | 'ENDED'     // The broadcast has concluded, and a recording should be available.
  | 'ABORTED'   // The broadcast was cancelled before or during its scheduled time.
  | 'RECORDING_FAILED'; // The broadcast ended, but the recording could not be processed.
```

---

### ClipStatus

An enumeration of possible states for a `BroadcastClip`.

```typescript
type ClipStatus =
  | 'PENDING_INGEST'      // The clip has been requested but media processing hasn't started.
  | 'INGESTED'            // The clip media has been cut and is available.
  | 'PENDING_ENRICHMENT'  // The clip is awaiting additional processing (e.g., transcription).
  | 'ENRICHED'            // Enrichment processes have completed.
  | 'PUBLISHED'           // The clip has been published to one or more platforms.
  | 'REJECTED'            // The clip was reviewed and deemed unsuitable for use.
  | 'ERROR';              // An error occurred during processing.
```

### AgentInstruction

Represents a task or command to be processed by an external agent or worker service.

```typescript
interface AgentInstruction {
  id: string;
  type: string; // A string identifying the type of instruction (e.g., 'PROCESS_CLIP', 'UPLOAD_PODCAST_EPISODE')
  payload: any; // The specific data pertinent to this instruction type
  createdAt: Date;
  targetAgentId?: string; // Optional: If the instruction is for a specific agent
  status: 'QUEUED' | 'IN_PROGRESS' | 'COMPLETED' | 'FAILED';
  attempts: number;
  lastAttemptedAt?: Date;
  error?: string;
}
```

**Example `AgentInstruction`:**

```json
{
  "id": "inst-12345",
  "type": "PROCESS_CLIP",
  "payload": {
    "clipId": "clip-abcde",
    "broadcastRecordingUrl": "https://example.com/recording.mp4",
    "startTime": 600,
    "endTime": 660,
    "outputFormat": "mp4"
  },
  "createdAt": "2023-10-27T10:00:00Z",
  "status": "QUEUED",
  "attempts": 0
}
```

### AgentAck

Represents an acknowledgment or response from an agent regarding a processed instruction.

```typescript
interface AgentAck {
  instructionId: string; // The ID of the instruction being acknowledged
  agentId: string;       // The ID of the agent processing the instruction
  status: 'SUCCESS' | 'FAILURE' | 'IN_PROGRESS'; // Outcome of the instruction processing
  message?: string;      // Optional: A descriptive message
  result?: any;          // Optional: Any data returned by the agent upon completion
  ackedAt: Date;
}
```

**Example `AgentAck`:**

```json
{
  "instructionId": "inst-12345",
  "agentId": "agent-processor-1",
  "status": "SUCCESS",
  "message": "Clip processed successfully.",
  "result": {
    "clipUrl": "https://cdn.example.com/clips/clip-abcde.mp4",
    "thumbnailUrl": "https://cdn.example.com/clips/clip-abcde.jpg"
  },
  "ackedAt": "2023-10-27T10:05:30Z"
}
```
