## TECHNICAL BLOCKERS (still valid — these are pipeline damage, not editorial)

These came in via the dictation/paste flow and got mangled by smart-quote autocorrect. Apply only to the locally-saved mockup; the web-session "final" may already be clean.

### T-1. CSS custom properties use en dashes, not double hyphens
Every `:root` variable is declared as `–bg`, `–ink`, etc. (one en dash). CSS requires `--bg`, `--ink` (two ASCII hyphens). **Result: zero variables resolve, every `var(--bg)` reference returns nothing, the page renders unstyled.**

```css
/* WRONG (what's in the file) */
:root { –bg: #FFFFFF; }
body { background: var(–bg); }

/* RIGHT */
:root { --bg: #FFFFFF; }
body { background: var(--bg); }
```

### T-2. Universal selector typed as a hyphen
```css
/* WRONG */ - { margin: 0; padding: 0; box-sizing: border-box; }
/* RIGHT */ * { margin: 0; padding: 0; box-sizing: border-box; }
```

### T-3. Font names wrapped in smart quotes
`font-family: ‘Fraunces’, …` should be `font-family: 'Fraunces', …` — straight ASCII single quotes.

### T-4. Article body wrapped in fenced code blocks
Each article's `<p>` content is bracketed with triple backticks. In an HTML document those render as literal characters above and below every story. **Strip the fence markers from all six articles.**
