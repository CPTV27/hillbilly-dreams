# Admin Documentation Voice

This is the canonical voice for all internal communication:

- Asana tasks
- Emails to partners
- Status updates
- Internal memos
- AI-generated drafts (Cos, Codex, GPT, etc.)

If it doesn't match this, it's wrong.

---

# Core principles

## 1. Lead with the action

Start with the verb. Not context.

Bad:
> We should probably look into...

Good:
> Close Visit Natchez next step

---

## 2. Be concrete, not abstract

Bad:
> Improve operational efficiency

Good:
> Reduce Inn check-in friction by standardizing arrival flow

---

## 3. No corporate language

Never use:
- circle back
- take it offline
- align on
- leverage (as a verb)
- synergy
- moving forward

If it sounds like a meeting, delete it.

---

## 4. Short > clever

Write like you're texting a smart person.

- No long paragraphs
- No filler
- No "tone"

---

## 5. Say what actually matters

Every piece of communication answers:

1. What is this?
2. Why does it matter?
3. What happens next?

If one is missing, rewrite.

---

## 6. Truth over polish

If something is uncertain, say it.

Bad:
> This will significantly improve results

Good:
> This might work. We'll know in a week.

---

# Standard formats

## Asana task format

```
[SOURCE] Verb + object

Why:
One sentence. Why this matters now.

Next:
The immediate next action.

Owner:
Suggested or assigned.

Due:
Realistic date.

Confidence:
High / Medium / Low
```

### Example

```
[Meeting] Close Adams County next step

Why:
Deal stalls without a defined next move.

Next:
Email Mitzi Conn with 2 concrete options and ask for call.

Owner:
Chase

Due:
Thursday

Confidence:
High
```

---

## Status update format

```
What shipped:
- X
- Y

What's in progress:
- X

What's blocked:
- X (why)

What I need:
- Specific ask
```

No narrative. No filler.

---

## Email format (internal)

```
One sentence opener (what this is)

Context (2–3 lines max)

Decision or action needed

Next step
```

### Example

```
Quick update on S2PX before Saturday.

We reviewed current state. Core issue is unclear ownership of deployment pipeline.

Decision:
Do we treat this as maintenance or rebuild?

Next:
If rebuild, we scope Monday. If maintenance, Elijah patches this week.
```

---

# Writing rules for AI agents

All agents (Cos, Codex, GPT, etc.) must:

1. Follow these formats exactly
2. Prefer brevity over completeness
3. Never invent certainty
4. Default to action-oriented outputs
5. Avoid summaries unless explicitly requested

---

# Anti-patterns (do not do this)

❌ Long explanations
❌ Repeating the same idea 3 ways
❌ "Here are some options" without a recommendation
❌ Vague tasks with no next step
❌ Emotional language in operational contexts

---

# The test

Before sending or saving anything, ask:

> Could someone act on this in 10 seconds?

If not, rewrite.

---

# Final rule

Clarity is the product.
Everything else is noise.
