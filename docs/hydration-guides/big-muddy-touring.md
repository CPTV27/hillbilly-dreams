### 4. Routing Strategy for `bigmuddytouring.com`

Since `bigmuddytouring.com` is the "default MBT hostname" and likely needs to serve content from the root (`/`), while other hostnames might serve different content (e.g., `app.bigmuddytouring.com` for the main application, or other white-labeled domains), we'll use a combination of Next.js Route Groups and Middleware.

This approach allows `bigmuddytouring.com` to serve its specific landing page from `app/(big-muddy)/page.tsx` while other requests (or hostnames) can be routed differently.

#### 4.1. Create a Route Group

1.  **Directory Structure:**
    Create a new route group `(big-muddy)` within your `apps/web/app` directory.
    ```
    apps/web/app/
    ├── (big-muddy)/
    │   ├── page.tsx          # The landing page content
    │   └── layout.tsx        # Optional: specific layout for bigmuddytouring.com
    ├── layout.tsx            # Root layout
    ├── page.tsx              # Default root page for other hostnames
    └── ...
    ```

2.  **Rationale:**
    *   Route groups (`(...)` folders) do not affect the URL path. So, `app/(big-muddy)/page.tsx` will still be accessible at `/`.
    *   They are useful for segmenting parts of your application, applying specific layouts, or, in this case, for middleware to target.

#### 4.2. Implement Middleware

Create a `middleware.ts` file at the root of your `apps/web` directory (`apps/web/middleware.ts`). This middleware will inspect the incoming hostname and rewrite the URL internally if it matches `bigmuddytouring.com`.

```typescript
// apps/web/middleware.ts
import { NextRequest, NextResponse } from 'next/server';

export const config = {
  matcher: [
    /*
     * Match all request paths except for the ones starting with:
     * - api (API routes)
     * - _next/static (static files)
     * - _next/image (image optimization files)
     * - favicon.ico (favicon file)
     * - any files in the public folder (e.g., /images, /docs)
     */
    '/((?!api|_next/static|_next/image|favicon.ico|images|docs|assets).*)',
  ],
};

export function middleware(req: NextRequest) {
  const url = req.nextUrl;
  const hostname = req.headers.get('host');

  // Define the target hostname for the Big Muddy Touring landing page
  const BIG_MUDDY_HOSTNAME = process.env.NEXT_PUBLIC_BIG_MUDDY_HOSTNAME || 'bigmuddytouring.com';
  const WWW_BIG_MUDDY_HOSTNAME = `www.${BIG_MUDDY_HOSTNAME}`;

  // Check if the current hostname matches bigmuddytouring.com
  if (hostname === BIG_MUDDY_HOSTNAME || hostname === WWW_BIG_MUDDY_HOSTNAME) {
    // If the path is already targeting the big-muddy group or is an API route,
    // let it pass to avoid infinite rewrites or breaking API calls.
    if (url.pathname.startsWith('/_next/') || url.pathname.startsWith('/api/')) {
        return NextResponse.next();
    }

    // Rewrite to the (big-muddy) route group.
    // This makes `bigmuddytouring.com/` internally serve `app/(big-muddy)/page.tsx`
    // and `bigmuddytouring.com/about` internally serve `app/(big-muddy)/about/page.tsx` (if it existed).
    // For this specific landing page, we only care about the root.
    if (url.pathname === '/') {
      return NextResponse.rewrite(new URL('/big-muddy', req.url));
    }
    // If there are other paths for bigmuddytouring.com, you might want to rewrite them too
    // e.g., return NextResponse.rewrite(new URL(`/big-muddy${url.pathname}`, req.url));
  }

  return NextResponse.next();
}
```
**Environment Variable:** Add `NEXT_PUBLIC_BIG_MUDDY_HOSTNAME=bigmuddytouring.com` to your `.env.local` or deployment environment.
